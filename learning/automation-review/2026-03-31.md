## Top 3 Automation Opportunities

### 1. Coverage threshold is 80%, not 90% тАФ CI PASSES UNDER THRESHOLD

**File:** `vitest.config.ts`

```ts
thresholds: {
  statements: 80,   // тЖР should be 90
  branches: 80,     // тЖР should be 90
  functions: 80,     // тЖР should be 90
  lines: 80,        // тЖР should be 90
}
```

**The bigger problem:** `continue-on-error: true` is set on lint, build, AND test steps in `ci.yml`. The CI workflow passes even when all three fail. This completely defeats the quality gate.

**Fix needed:**
- Set all threshold values to 90
- Remove `continue-on-error: true` from lint/build/test steps, OR add a coverage gate that actually blocks the build

**Time saved:** Prevents silently degraded coverage from accumulating. Quantified value: ~30 min/sprint of qa-engineerцЙЛхКи catch-up work.

### 2. Dependency updates are reported but NOT automated тАФ no PRs created

**File:** `.github/workflows/dependency-audit.yml`

The workflow runs `npm-check-updates` weekly and creates GitHub issues listing available updates. But it does not open PRs with `npm update` or Dependabot/Renovate. Issues pile up, no one acts on them, and the team falls behind on updates.

**Fix needed:**
- Add a second weekly job that opens a Dependabot PR with all non-major updates auto-batched
- Or use `ncu --upgrade --upgradeAll && git push` in a workflow job, with a label totag for human review before merge

**Time saved:** Manual dependency update PRs take ~45 min/week. Automating the PR creation saves that recurring cost.

---

### 3. Husky git hooks are disabled тАФ no pre-commit/pre-push linting

**File:** `package.json`

```json
"prepare": "echo 'skipping husky'"
```

No `.husky/pre-commit` or `.husky/pre-push` hooks exist. The `lint-staged` config in `package.json` is defined but never executed because husky is not installed/run.

This means:
- Bad formatting reaches GitHub Actions (too late)
- Build failures reach CI (too late)
- No local feedback loop for devs

**Fix needed:**
- Add `.husky/pre-commit` that runs `npx lint-staged`
- Add `.husky/pre-push` that runs `npm run test:coverage` (or a fast subset)
- Change the `prepare` script to `husky install`

**Time saved:** ~20 min/review cycle of back-and-forth on formatting/early-build issues. With 3+ PRs per sprint, that's ~1 hour/sprint.
