# FamilyTV QA Sprint Report — 2026-04-01

## Issue: Deploy Gate Blocking PR #35 (watch-party)

### Root Cause: Missing `coverage/coverage-summary.json`

**Symptom:** Deploy gate `coverage` job fails at the "Enforce 97% coverage" step because it cannot find `coverage/coverage-summary.json`.

**Deploy Gate expects:** File at `coverage/coverage-summary.json` with format like:
```json
{"total": {"lines":{"pct":97.32,...},...}}
```
The deploy gate grep pattern is: `grep -o '"lines":{"pct":[0-9.]*'`

**What vitest actually produces:** `coverage/coverage-summary.json` does NOT exist. The current `vitest.config.ts` coverage reporters are:
```ts
reporter: ["text", "text-summary", "lcov"]
```
Missing `"json-summary"` reporter. No JSON file is generated.

---

## Fix #1 (vitest.config.ts)

**Change:**
```ts
coverage: {
  provider: "v8",
  reporter: ["text", "text-summary", "lcov"],
  // ...
}
```

**To:**
```ts
coverage: {
  provider: "v8",
  reporter: ["text", "text-summary", "json-summary", "lcov"],
  // ...
}
```

**Why:** The `"json-summary"` reporter produces `coverage/coverage-summary.json` in the exact format the deploy gate expects: `{"total":{"lines":{"pct":...},...}}`. Verified locally — when run with `--coverage.reporter="json-summary"`, the file is generated correctly with 99.38% lines / 97.32% branches.

## Fix #2 (vitest.config.ts) — Covert Coverage Inflation Issue

### The Problem

The `coverage.include` whitelist in `vitest.config.ts` is **artificially inflating coverage numbers** by only measuring a curated subset of files:

```ts
include: [
  "src/lib/utils.ts",
  "src/app/api/posts/route.ts",
  "src/app/api/comments/route.ts",
  "src/app/api/reactions/route.ts",
  "src/app/api/family/route.ts",
  "src/app/api/family/activity/route.ts",
  "src/app/api/invite/route.ts",
  "src/app/api/events/route.ts",
  "src/components/progress-dots.tsx",
  "src/components/theme-toggle.tsx",
  "src/components/error-boundary.tsx",
  "src/components/notifications-client.tsx",
],
```

This means the vast majority of the codebase is **excluded from coverage entirely**. The 97.32% branch figure in CI only applies to this whitelist.

### Local vs CI Discrepancy Explained

| Environment | Branch Coverage | Reason |
|---|---|---|
| Local (86%) | Likely running with wider include (or default include), covering more files including ones with uncovered branches | Realistic view |
| CI (97.32%) | Only the curated whitelist is measured | Artificially inflated by exclusion |

### Recommended Fix

Either:

**Option A (recommended):** Remove the `include` whitelist entirely and let vitest measure all source files. Adjust the `thresholds` accordingly (97% may be too aggressive for a real codebase — industry standard is often 80%).

**Option B:** If the whitelist is intentional (only caring about specific critical paths), document this clearly and add a comment explaining why most of the codebase is excluded. Also consider whether the 97% threshold is the right number for only those files.

**Option C:** Expand the whitelist to include ALL source files that should be tested, not just a curated subset.
