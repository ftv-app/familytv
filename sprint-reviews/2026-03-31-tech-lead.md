### 3. New Test: `returns 400 for video post with non-string mediaUrl` ✅

**Test purpose:** Validates that video posts with a non-string `mediaUrl` (e.g., `12345`) receive a 400 response.

**Implementation validation:**
```typescript
// posts/route.ts:
if (contentType !== "text" && (!mediaUrl || typeof mediaUrl !== "string")) {
  return NextResponse.json(
    { error: "mediaUrl is required for video/image posts" },
    { status: 400 }
  );
}
```

When `contentType: "video"` and `mediaUrl: 12345`:
- `contentType !== "text"` → `true`
- `!mediaUrl` → `false` (12345 is truthy)
- `typeof mediaUrl !== "string"` → `true`
- Combined condition: `true && true` → triggers 400 ✅

**Verdict:** ✅ Correct and important edge case. Type coercion issues like this are easy to miss without explicit tests.

### 4. New Test: `uses 'Family member' fallback when auth has no fullName or firstName` ✅
